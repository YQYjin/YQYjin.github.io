1.什么是代数（algebra）?

​ 为了形式化一个概念，构建出有关这个概念的符号以及操作符号的公式。

2.什么是线性代数（linear algebra）？

​ 一项关于向量以及操作向量的公式的研究

3.什么是线性方程组（system of linear equation）？

​ 其中，是这个式子的未知量，每一个满足这个方程组的n元组都可以称为线性方程组的一个解。

5.实数线性方程组的解（solution）有哪些情况？

​ 要么无解，要么有一个解，要么有多个解。

​ 如果是，那么可以看做二维平面两条直线相交，那么结果是1.无交集2.一个点3.一条直线。如果是，那么可以看做是在三维立体空间两个平面相交，结果可能是1.一个平面2.一条直线3.一个点4.空。

6.什么是点积（dot product），也称作Hadamard product?

​ 对于元素，我们用矩阵A的第i行与矩阵B的第j列逐个元素相乘，然后相加。

7.什么是单位矩阵（Identity Matrix）？

​ 在n阶方阵中，只是在对角（diagonal）元素值为1，在其他元素值为0。

8.什么是矩阵的逆（inverse）？

​ 对于n行n列的矩阵A和n行n列的矩阵B，如果满足 AB=I=BA，B被叫做A的逆。如果B存在，那么称A为正则矩阵/可逆矩阵/非奇异矩阵（regular/invertible/nonsingular）。如果矩阵的行列式（determinant）不为零，那么可以认为矩阵可逆。

9.什么是矩阵的转置（transpose）？

​ 对于m行n列的矩阵A，A的转置B是n行m列，A的每一行作为B的每一列。如果A可逆，那么B（A的转置）爷可逆。

10.什么是高斯消元法（Gaussian elimination）

​ 高斯消元法是通过初等变换（elementary transformation）把一个线性方程组的增广矩阵（augmented matrix），譬如，那么其增广矩阵为，变成一个行递减的形式（reduced row-echelon）。

11.行递减阶梯矩阵有什么特点？

​ 是一个行阶梯的形式
​ 每个枢轴的值都是1
​ 每一列只有唯一一个枢轴是这一列不为零的值

DropMessage(人工智能导论)

发表于 2023-11-12 更新于 2023-11-14 分类于 课程

DropMessage-GNNs

1.摘要

  • GNN存在过拟合,过平滑和非稳健性的问题,这些问题可以通过随机丢弃的方法缓解.
  • 提出DropMessage作为新型随机丢弃方法,在消息传递过程中直接对预控消息执行丢弃操作,为目前的随机丢弃方法提供了一个统一的框架.
  • DropMessage通过降低样本方差来稳定训练过程,从信息论的角度保持了信息多样性.

2.导言

  • GNN与其他数据模型相比,图数据标签繁重昂贵且存在偏差,由于过拟合限制了GNN的泛化能力,且过渡平滑导致GNN无法有效模拟来自多跳邻居间的高阶依赖关系.
  • GNN的递归聚合模式使其易受输入图的质量影响.
  • 随机丢弃法可以缓解上述问题,该方法通过随机屏蔽输入的内容,将增强数据整合到模型中.
  • 随机丢弃法存在问题:增强数据会导致参数难以收敛,训练过程不稳定,且难以找到一种普遍的丢弃率
  • DropMessage
    • 与传统随机丢弃在节点特征矩阵上丢弃不同,DropMessage在传播的消息上丢弃,使一个节点可以向不同邻居传播不同的消息.
    • 证明了在GNN上使用随机丢弃等于在损失函数中引入额外的正则化项,使模型更加稳健.
    • DropMessage样本方差更小,训练更稳定,从信息论的角度保持了信息的多样性,理论上是其他丢弃方法的上线.
    • 将其他丢弃方法统一到DropMessage框架中,使其他方法可以看作是DropMessage的一种特殊情况.

3.相关工作

  • 随机丢弃有三种最常见的方法:Dropout(丢弃矩阵中的某些信息),DropEdge(丢弃矩阵的一列或一行),DropNode(丢弃一个节点)
  • 然而,它们只能在特定的数据集和GNN模型上有好的效果,如何找到一个普遍适用的随机丢弃方法依旧是一个问题.

4.DropMessage

  • 该算法直接对信息矩阵进行操作,而不是对特征矩阵进行操作(传统的)

  • 该算法仅对训练过程有影响

  • DropNode,DropEdge,DropOut都是丢弃特征矩阵中的元素来影响信息矩阵,而DropMessage直接对信息矩阵进行操作,最为精细,使其成为了最灵活的丢弃方式,而其他丢弃方式可以看作是DropMessage的一种特殊形式

5.DropMessage的优势

5.1 提高训练的稳定性

  • 与其他方法相比,DropMessage通过减少训练样本的方差来提高训练的稳定性
  • 通过减少样本方差,减少了不同训练时期之间消息矩阵的差异,从而稳定训练过程并加速收敛

5.2保持信息多样性

  • 信心多样性包括特征多样性和拓扑多样性,只有在随机丢弃后特征多样性和拓扑多样性都不降低时,才认为具有保持信息多样性的能力

  • DropOut,DropNode,DropEdge都不能保持信息多样性,因为它们都对特征矩阵进行丢弃,前者导致特征多样性降低,而后两者导致特征多样性和拓扑多样性都降低

  • 保持信息多样性的随机丢弃算法可以保持更多的信息,因此理论上没有更优于DropMessage的方法.即使DropMessage会丢失一定的信息,它的信息保有量依旧高于其他两种方法.

Redis速成

发表于 2023-09-14 更新于 2023-09-16 分类于 工程

一.安装配置

  • 使用sudo service redis-server start 启动Redis服务器,使用redis-cli启动客户端
  • 推荐使用RedisInsight操作Redis
  • 切换WSL版本:wsl --set-version Ubuntu-版本号 2

二.数据类型

1.字符串

  • Redis中的数据都是以键值对存在,使用SET命令来创建数据,Redis中区分大小写

  • SET name redis即创建了一个键值对: name:redis

  • Redis中默认使用字符串存储数据且是二进制安全的

  • 使用DEL命令删除键值对(删除键及对应的值)

  • 使用EXISTS命令查看某个键是否存在,使用KEYS 来查找键,使用FLUSHALL删除所有键

  • Redis默认不支持中文显示(显示的是二进制),在登录时使用redis-cli --raw来正常显示中文

  • 使用clear清空屏幕

  • 使用TTL来查看键的过期时间,使用EXPIRE 键 过期时间来设置过期时间

  • SETNX创建值,当键不存在时创建,当键已存在时不执行任何操作

2.列表List

  • 使用LPUSH向列表中添加数据,使用LRANGE命令从列表中获取数据
  • LPUSH是将数据添加到列表头部,RPUSH将数据添加到列表尾部
  • LPOP和RPOP用于删除元素
  • LTRIM删除区间之外的所有元素

3.SET

  • SET是无序的,不能有重复元素,SET相关命令都以S开头
  • SADD向集合中添加元素.SMEMBERS查看集合中的元素,SREM删除集合中的元素
  • SINTER,SUNION等命令用于集合间的运算

4.有序集合

​ 有序集合会将集合中的元素安装从小到大排序

  • ZADD用于添加元素,ZRANGE 查询集合中的键,ZSCORE查看集合中的值,ZRANK查看键的排名
  • 使用ZREVRANK反转集合,使集合从大到小排序
  • ZREM删除,等等其他命令

5.哈希

​ HASH相关命令都以H开头

  • HSET添加键值对,HGET获取键值对,HGETALL获取所有的键值对
  • HDEL删除键值对,HEXISTS判断键值对是否存在,HKEYS获取所有键,HLEN获取键值对的数量
  • 以及其他更多命令

三.发布订阅模式

  • 使用subscribe +名称来订阅一个频道,就可以接收该频道的消息

  • 使用publish+名称来向频道中发送消息,所有订阅该频道的客户端都可接收

  • 使用频道无法使消息持久化,无法查看历史消息

四.Stream

​ Stream相关消息都以X开头,可以解决频道无法持久化数据和查看历史数据的问题

  • 使用XADD 添加消息,使用XLEN查看消息数量,使用XRANGE查看消息内容

  • 使用XDEL删除消息,XTRIM批量删除消息

  • 可以使用ID发布消息(可手动指定或Redis自动生成)

  • XREAD读取消息,可以设置读取的消息数量,没有消息时阻塞等

  • 还可以创建一个消费者组XGROUP,XINFO查看消费者组信息,XGROUP CREATECONSUMER向消费者组中添加消费者,XINFO GROUPS查看消费者组信息,可以设置阻塞等

五.其他数据结构

1.地理空间Geospatial

​ Redis3.2中的新特性,是一种存储地理位置信息的数据结构,支持地理位置直接的计算,相关指令都以GEO开头

  • GEOADD添加地理位置信息,位置信息是经纬度
  • GEOPOS获取某个地点的位置,GEODIST计算两个位置直接的距离,单位默认为米
  • SEARCH搜索指定范围内的地理位置并返回,范围中心可以是地点或经委度,范围可以是圆形也可以是矩形

2.HpyerLogLog

​ 是一种用于统计基数的算法,基数:不重复元素的格式(不确定),使用随机算法减少性能消耗,因此存在误差

  • PFADD向集合中添加数据,PFCOUNT统计基数,PFMERGE合并集合

3.位图Bitmap

​ 位图是字符串类型的拓展,应用范围广泛,用于统计是否点赞等

  • SETBIT设置位图,GETBIT获取某个偏移量的值
  • 使用SET批量设置,如使用16进制表示2进制,一次性设置多个偏移量
  • BITCOUNT统计位图中有几个bit值为1
  • BITPOS获取位图中第一个出现的1或0的位置

4.位域

​ 位域能将很多小的整数存储到一张大的位图中,以提高内存利用率

六.多操作

  • 使用MULTI开启多操作,使用EXEC执行多操作中的所有命令

  • 在Redis中,在发送EXEC命令之前,所有命令都会被放入队列中缓存,不被立即执行;事务中的某一指令执行失败,其他指令依旧会被执行;事务执行的过程中其他客户端提交的命令请求不会被插入到事务中

七.持久化

  • RDB持久化
    • 在指定时间间隔,将数据快照写入磁盘
    • 使用SAVE命令手动写入,使用BGSAVE创建一个子进程写入数据
    • 最后一次快照之后到下一次快照之间的数据会丢失
  • AOF持久化
  • 在执行写命令时,将日志写入AOF,重启数据库中重新操作AOF中的命令来恢复数据库

八.主从复制

  • 一个主节点有多个从节点,一个从节点只有一个主节点

九.哨兵模式

​ 实现自动故障转移,哨兵以独立的进制运行在Redis集群中,监控各个节点

  • 监控:不断发送命令检查Redis节点是否正常
  • 通过:通过消息订阅发布节点故障信息
  • 自动故障转移:主节点故障时,将某个子节点升级为主节点

1.准备工作

1.1 Gorm框架安装

​ 使用Go自带的get进行安装,首先打开命令行(CMD或Shell),将路径切换至项目路径(go.mod所在路径),依次执行以下命令:

  • go get -u gorm.io/gorm
  • go get -u gorm.io/driver/mysql

​ 即可安装Gorm框架和Gorm的Mysql驱动,本文使用Mysql作为数据库,若使用其他数据库,请下载相应数据库驱动

1.2 DSN

​ Gorm框架的驱动使用DSN来连接数据库,因此需要配置DSN,DSN连接格式如下:

  • user:password@tcp(127.0.0.1:3306)/database_name?charset=utf8mb4&parseTime=True&loc=Local

​ 分别是:用户名,密码,服务器地址(127.0.0.1),端口(3306),数据库名称等

​ 首次配置方法(Windows):

  • 打开控制面板->管理工具->ODBC数据源->系统DSN
  • 添加MySQL ODBC 8.0 Driver
  • 设置数据源名称(随便命个名),TCP/IP 服务器地址(建议127.0.0.1:3306)
  • 输入用户名和密码,MySQL存在的用户,要有对应的表权限,若输入正确则可以选择下一列的数据库名称

对应的DSN为: root:123456@tcp(127.0.0.1:3306)/bloginfos?charset=utf8mb4&parseTime=True&loc=Local

3.查询

3.1 查询方式

​ Gorm提供三种方式来查询一条单一数据,分别是:

  • First查询第一条数据(主键递增的第一条数据)
  • Take查询一条数据,无顺序
  • Last查询最后一条数据(即主键递减的第一条数据)

​ Gorm通过Find来查询所有符合条件的元组

​ 注意参数要传递引用,因为Gorm通过修改参数来返回结果,若不传递引用则无法得到结果

​ 对于没有设置主键的表,则按第一个字段排序

​ 这三种方法可以设置查询条件,如下所示,对于Find方法,查询到的所有结果会以数组的方式存储在uesr中

​ 也可以不指定对象类别,适用表名来进行查询

3.2 条件查询

​ Gorm有多种方法设置查询条件,此处以First为例,条件设置方式适用于其他查询方法

​ 查询条件都以字符串形式出现,为查询字段+值的形式设置,其中查询字段为: 字段名+约束方法+?的形式组成,约束方法可以设置为SQL中的方法如=,>,<,between,in等

  • 主键查询:db.First(&user, 1)查询主键值为1的元组

  • 单一条件查询:db.First(&user, "username = ?", "admin")查询用户名为admin的元组

  • 多条件查询:db.First(&user, []string{"username = ?", "admin", "password = ?", "123456"})

​ 除了上述查询条件的设置方法,还有以下几种:

  • db.Where("name = ?", "jinzhu").First(&user)
  • db.Where(&User{Name: "jinzhu", Age: 20}).First(&user)
  • db.Where(map[string]interface{}{"name": "jinzhu", "age": 20}).Find(&users)

​ 也可以在一条字符串中设置多个查询条件,根据条件的数量依次读取后续参数作为查询值

  • db.Find(&users, "name <> ? AND age > ?", "jinzhu", 20)

6.删除

​ Gorm中提供了物理删除和软删除两种删除方法,物理删除将直接从数据库中删除该数据,而软删除会设置deleted_at字段而不会在数据库中删除,软删除后该条数据无法通过Gorm一般查询被查询到

6.1物理删除

​ 若不指定条件,则按实例的主键作为条件删除对应数据,或与Where搭配使用

​ 不传递实例,按主键删除

​ 批量删除,不传递实例

6.2 软删除

​ 若表中有gorm.DeletedAt字段,则该表可以使用软删除(该字段包含在gorm.Model中),软删除会被自动调用

​ 软删除的原理是执行Update来为DeletedAt字段赋值为执行删除命令的时间

  • db.Delete(&user)// UPDATE users SET deleted_at=删除时间 WHERE id = user.id;

​ 如果不想使用软删除,可以使用Uncoped执行下述命令

  • db.Unscoped().Delete(&order) // DELETE FROM orders WHERE id=10;
