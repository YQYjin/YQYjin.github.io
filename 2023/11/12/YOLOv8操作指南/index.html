readme

官方源码中的readme路径

模型下载

类别

​ 对于不同的任务,模型的内容不一样,一般根据模型名称来区分模型,执行任务时,YOLO会尝试根据模型来确定任务类别

名称 模型文件 家族
检测 yolov8n.pt 8n、8s、8m、8l、8x
分割 yolov8n-seg.pt 8n、8s、8m、8l、8x
分类 yolov8n-cls.pt 8n、8s、8m、8l、8x
姿态 yolov8n-pose.pt 8n、8s、8m、8l、8x

2.训练及预测

启动预测

​ model:设置模型文件; soruce:预测图片所在的文件夹(路径)

​ 可以选择多种预测模式,包括classify,detect,pose和segment,使用对应的预测器即可,如detect为DetectionPredictor

启动训练

使用YOLO构建模型

  • .yaml文件,则是从yaml中构建一个新模型,从零开始训练model=YOLO('yolov8n.yaml')
  • .pt文件,加载与训练模型model = YOLO('yolov8n.pt')

向train()中添加训练参数来自定训练,需要指定参数名=参数值

训练参数

YOLO模型的训练设置是指用于在数据集上训练模型的各种超参数和配置。这些设置可以影响模型的性能、速度和准确性。一些常见的YOLO训练设置包括批大小、学习率、动量和权重衰减。可能影响训练过程的其他因素包括优化器的选择、损失函数的选择以及训练数据集的大小和组成。重要的是要仔细调整和尝试这些设置，以实现给定任务的最佳性能。

键 值 描述
model None 模型文件路径，例如 yolov8n.pt，yolov8n.yaml
data None 数据文件路径，例如 coco128.yaml
epochs 100 训练的时代数
patience 50 用于提前停止训练的观察不到改善的时代数
batch 16 每批次的图像数量（-1 为自动批处理）
imgsz 640 输入图像的大小（整数形式）
save True 保存训练检查点和预测结果
save_period -1 每 x 时代保存一次检查点（如果小于 1，则禁用）
cache False True/ram、disk 或 False。使用缓存进行数据加载
device None 运行的设备，例如 cuda device=0 或 device=0,1,2,3 或 device=cpu
workers 8 数据加载的工作线程数（如果使用 DDP，则每个 RANK 一个线程）
project None 项目名称
name None 实验名称
exist_ok False 是否覆盖现有实验
pretrained True （布尔或字符串）是否使用预训练模型（布尔）或从中加载权重的模型（字符串）
optimizer 'auto' 要使用的优化器，选项=[SGD, Adam, Adamax, AdamW, NAdam, RAdam, RMSProp, auto]
verbose False 是否打印详细输出
seed 0 用于可重现性的随机种子
deterministic True 是否启用确定性模式
single_cls False 将多类数据训练为单类
rect False 每批次进行矩形训练，以最小填充
cos_lr False 使用余弦学习率调度器
close_mosaic 10 （整数）在最后时期禁用马赛克增强（0 禁用）
resume False 从上次检查点恢复训练
amp True 自动混合精度（AMP）训练，选项=[True, False]
fraction 1.0 训练的数据集部分（默认为 1.0，所有训练集中的图像）
profile False 在训练期间为日志记录的 ONNX 和 TensorRT 速度启用分析
freeze None （整数或列表，可选）冻结前 n 层，或在训练期间冻结的层索引列表
lr0 0.01 初始学习率（例如，SGD=1E-2，Adam=1E-3）
lrf 0.01 最终学习率（lr0 * lrf）
momentum 0.937 SGD 动量/Adam beta1
weight_decay 0.0005 优化器权重衰减 5e-4
warmup_epochs 3.0 预热时代数（小数也可行）
warmup_momentum 0.8 预热时的初始动量
warmup_bias_lr 0.1 预热时的初始偏置学习率
box 7.5 盒损失增益
cls 0.5 分类损失增益（与像素成比例）
dfl 1.5 dfl 损失增益
pose 12.0 姿势损失增益（仅姿势）
kobj 2.0 关键点对象损失增益（仅姿势）
label_smoothing 0.0 标签平滑（分数）
nbs 64 名义批大小
overlap_mask True 训练期间掩码应该重叠（仅分割训练）
mask_ratio 4 掩码降采样比例（仅分割训练）
dropout 0.0 使用 dropout 正则化（仅分类训练）
val True 在训练期间进行验证/测试

3.评估

  • 使用python进行评估
  • 评估时可以自定一些参数,参数放入.val()中,可选参数列表如下:

​ YOLO模型的验证设置涉及用于评估模型在验证数据集上性能的各种超参数和配置。这些设置可能会影响模型的性能、速度和准确性。一些常见的YOLO验证设置包括批处理大小、训练期间执行验证的频率以及用于评估模型性能的指标。可能影响验证过程的其他因素包括验证数据集的大小和组成，以及模型正在执行的具体任务。重要的是要仔细调整和尝试这些设置，以确保模型在验证数据集上表现良好，并检测和防止过拟合。

键 值 描述
data None 数据文件路径，例如 coco128.yaml
imgsz 640 输入图像的大小，以整数表示
batch 16 每批图像的数量（-1表示自动批处理）
save_json False 将结果保存到JSON文件
save_hybrid False 保存标签的混合版本（标签 + 额外预测）
conf 0.001 用于检测的对象置信度阈值
iou 0.6 非极大值抑制（NMS）的交并比阈值
max_det 300 每个图像的最大检测数
half True 使用半精度（FP16）
device None 运行设备，例如 cuda device=0/1/2/3 或 device=cpu
dnn False 使用OpenCV DNN进行ONNX推断
plots False 在训练过程中显示图表
rect False 每批进行矩形验证，以实现最小填充
split val 用于验证的数据集拆分，例如 ‘val’、’test’ 或 ‘train’

4.推理

  • 使用python进行推理
  • 可用参数

model.predict()接受多个参数，这些参数可以在推断时传递以覆盖默认设置：

名称 类型 默认值 描述
source str 'ultralytics/assets' 图像或视频的源目录
conf float 0.25 用于检测的对象置信度阈值
iou float 0.7 非极大值抑制（NMS）的交并比阈值
imgsz int 或 tuple 640 图像大小，可以是标量或(h, w)列表，例如(640, 480)
half bool False 使用半精度（FP16）
device None 或 str None 运行设备，例如 cuda device=0/1/2/3 或 device=cpu
show bool False 如果可能，显示结果
save bool False 保存带有结果的图像
save_txt bool False 将结果保存为 .txt 文件
save_conf bool False 带有置信度分数保存结果
save_crop bool False 带有结果的裁剪图像保存
hide_labels bool False 隐藏标签
hide_conf bool False 隐藏置信度分数
max_det int 300 每个图像的最大检测数
vid_stride bool False 视频帧速率步长
stream_buffer bool False 缓冲所有流式帧（True）或返回最新的帧（False）
line_width None 或 int None 边界框的线宽。如果为None，则按比例缩放到图像大小
visualize bool False 可视化模型特征
augment bool False 对预测源应用图像增强
agnostic_nms bool False 类别无关的NMS
retina_masks bool False 使用高分辨率分割掩码
classes None 或 list None 按类过滤结果，例如 classes=0 或 classes=[0,2,3]
boxes bool True 在分割预测中显示边界框
